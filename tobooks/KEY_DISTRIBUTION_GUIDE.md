# 密钥分发指南

## 🎯 密钥分发策略

### 高级用户组 (20个私钥)
**适用对象**: VIP用户、付费用户、重要客户
**分发方式**: 
- 每人分配 1-2 个私钥（主用+备用）
- 可用于多设备同时登录
- 永久有效，优先技术支持

**使用场景**:
```
张三 (VIP用户) → private_1.pem + private_2.pem (备用)
李四 (企业客户) → private_3.pem  
王五 (长期用户) → private_4.pem + private_5.pem (备用)
...
```

### 标准用户组 (50个私钥)
**适用对象**: 普通注册用户、活跃用户
**分发方式**:
- 每人1个私钥
- 支持多设备使用
- 永久有效

**使用场景**:
```
用户A → private_1.pem
用户B → private_2.pem  
用户C → private_3.pem
...可支持50个标准用户
```

### 试用用户组 (100个私钥)
**适用对象**: 新用户、临时用户、推广用户
**分发方式**:
- 批量分发给新用户
- 可用于推广活动
- 永久有效，但优先级较低

**使用场景**:
```
新用户注册 → 自动分配 trial_users_X.pem
推广活动 → 批量发放试用私钥
临时访客 → 提供试用账号
```

## 🔐 安全机制

### 公钥作用
- **白名单验证**: 只有对应公钥的私钥才能通过认证
- **防伪造**: 无法伪造有效的数字签名
- **批量管理**: 可以批量禁用某组用户

### 私钥管理
- **用户持有**: 私钥由用户保管，服务器不存储
- **多设备使用**: 同一私钥可在多设备使用
- **永久有效**: 认证一次，永久访问
